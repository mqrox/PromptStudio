# Prompt Skill PDCA

## なぜ保存するのか

PromptStudio では、生成されたプロンプト本文だけを保存しても不十分である。

重要なのは、以下が後から分かること。

- ユーザーは何を作りたかったのか
- どの会話で意図が具体化されたのか
- どのskillが使われたのか
- どのknowledgeが参照されたのか
- どのモデルや生成サービスを想定したのか
- 管理者が何を改善すべきと判断したのか

これらが残ることで、個人の試行錯誤が会社のskill改善につながる。

## skillとは何か

PromptStudio における skill は、単なるプロンプトテンプレートではない。

skill は、以下を含む実務ノウハウの単位である。

- 画像生成向けの意図分解
- 動画生成向けの時間、動き、カメラ指定
- 広告、映像、CG / VFX、デザイン制作で必要な画作りの観点
- モデルごとの得意不得意
- ネガティブプロンプトの考え方
- 案件ルールや禁止事項
- レビューでよく指摘される失敗パターン
- 出力を再現しやすくする記述順序

## knowledgeとは何か

knowledge は、案件や会社で共有する文脈である。

例。

- 案件ごとのトーン、世界観、禁止事項
- ブランドガイドライン
- キャラクター設定
- 用語集
- 参照作品の分析メモ
- モデル別の使い方
- 社内レビュー基準

skill が「どう考えるか」だとすると、knowledge は「何を前提にするか」である。

## 保存する成果物の例

1回の相談結果は、以下のようなフォルダーで保存する。

```text
artifacts/
  2026-07-08_153000_project-shot-idea/
    request.md
    conversation.md
    prompt.md
    prompt.json
    references.md
    review.md
```

### request.md

ユーザーの最初の依頼、目的、用途、制約を記録する。

### conversation.md

対話の流れを記録する。

### prompt.md

人間が読みやすい最終プロンプトを記録する。

### prompt.json

機械処理しやすい構造化データを記録する。

### references.md

参照したskill、knowledge、workspace内のファイルを記録する。

### review.md

管理者がレビューし、skill改善へ戻すためのメモを記録する。

## PDCA

### Plan

会社として使うskillとknowledgeを用意する。

### Do

制作スタッフがChrome拡張で相談し、プロンプトと副産物を保存する。

### Check

管理者が保存物をレビューし、品質、再現性、モデル適性、案件適合性を確認する。

### Act

レビュー結果をskill / knowledgeへ反映し、次回以降の出力を改善する。

## レビュー観点

管理者は、少なくとも以下を見る。

- 制作意図を正しく分解できているか
- 画像生成 / 動画生成の用途に合っているか
- モデルの制約に合っているか
- 不要に曖昧な表現が残っていないか
- 社内ルールや案件ルールに違反していないか
- 次回も再利用できる形になっているか
- skillに戻すべき学びがあるか
