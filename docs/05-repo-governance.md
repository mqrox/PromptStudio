# Repository Governance

## このリポジトリの役割

PromptStudio の設計、実装、運用ルールを一元管理する。

このリポジトリは、単なるChrome拡張のコード置き場ではない。会社の生成AIプロンプト運用を成立させるための、skill、knowledge、保存形式、レビュー運用も含めて管理する。

## ディレクトリ方針

```text
docs/
  設計、MVP、運用、意思決定

extension/
  Chrome拡張

bridge/
  Chrome拡張とローカルworkspaceをつなぐ処理

skills/
  会社共通のプロンプトskill

knowledge/
  共有コンテキストのサンプルと構造

examples/
  保存される成果物の例
```

## ドキュメント方針

- ユーザー、管理者、実装者が読む文書は日本語で書く
- 実装の細部より、なぜその設計にしたかを残す
- 重要な設計変更は `docs/06-decision-log.md` に追記する
- 既存方針と矛盾する変更を入れる場合は、理由を明示する

## 実装方針

- Chrome拡張とローカルブリッジを分離する
- 保存形式を先に安定させる
- 特定のAIサービス、IDE、モデルに密結合しない
- APIキーや個人情報をコミットしない
- MVPでは安全なローカル動作を優先する

## Git運用

- `main` を現在の基準ブランチとする
- まとまった設計変更、実装変更ごとにコミットする
- コミットメッセージは変更内容が分かる短い英語または日本語にする
- 生成物やローカル成果物は原則コミットしない

## レビュー観点

変更時は以下を確認する。

- 現場スタッフが使いやすくなるか
- 管理者レビューとskill改善につながるか
- Chrome拡張に過剰なローカル権限を持たせていないか
- 保存形式が後から検索、レビュー、再利用しやすいか
- Google Antigravity / Antigravity IDE 以外にも将来差し替えられる余地があるか
