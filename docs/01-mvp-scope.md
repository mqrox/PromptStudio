# MVP Scope

## MVPの目的

最初のMVPでは、VFX / CG 制作スタッフが Chrome 上で対話しながら、画像生成 / 動画生成向けのプロンプトを作り、その過程と結果をローカルworkspaceに保存できる状態を目指す。

完成度の高い管理画面よりも、まずは以下を優先する。

- 社員が迷わず相談できる
- 会社共通のskill / knowledgeを参照できる
- 生成プロンプトと副産物が保存される
- 管理者が後からレビューできる
- レビュー結果をskill改善に戻せる

## MVPで作るもの

### Chrome拡張

- サイドパネル形式のチャットUI
- 日本語での対話入力
- 画像生成 / 動画生成 / その他の用途選択
- 生成プロンプトの表示
- 保存ボタン
- 設定画面

### 設定

- ローカルworkspaceのパス
- skillフォルダーのパス
- knowledgeフォルダーのパス
- 成果物保存先フォルダーのパス
- ユーザー識別名
- 将来のGoogleログインに備えたユーザーID欄

### 保存物

1回の相談単位で、以下を保存する。

- ユーザーの依頼内容
- 対話ログ
- 生成されたプロンプト
- ネガティブプロンプト
- 推奨設定
- 参照したskill
- 参照したknowledge
- モデル / サービス想定
- 管理者レビュー用メモ

### ローカルブリッジ

Chrome拡張から任意フォルダーに直接保存するのではなく、ローカルブリッジを経由して保存する。

ローカルブリッジは、MVPでは最小限のlocalhost APIまたはNative Messaging hostとして扱う。

## MVPで後回しにするもの

- Google Workspace連携
- 本格的な権限管理
- チーム別ダッシュボード
- クラウド同期
- 生成AIサービスへの直接投稿
- 画像や動画の自動評価
- 複数拠点の集中管理

これらは重要だが、最初に作るべき核ではない。

## 成功条件

MVPが成功している状態は以下。

- 制作スタッフがChrome上で自然に相談できる
- 出てきたプロンプトが画像生成 / 動画生成で使える構造になっている
- 保存されたフォルダーを見れば、何を作ろうとして、どのプロンプトになったか分かる
- 管理者が保存物を見て、skillの改善点を指摘できる
- 同じskill改善が次回以降の出力に反映できる
