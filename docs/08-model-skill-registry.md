# Model Skill Registry

## 目的

PromptStudio は、仕事でクリエイティブ制作の生成AIプロンプトを考えるときに最初に使う共通ワークベンチである。

そのため、1つのAIモデルだけに最適化しない。意図、用途、媒体、入力素材、使うモデルに応じて、必要なskillを呼び出せるようにする。

モデル別skillは、プロンプトを固定化して全員の出力を平均化するためのものではない。各クリエイターの意図や個性を保ちながら、モデルに渡す前提条件、入力素材の扱い方、失敗しやすい条件を揃えるために使う。

## 初期対応候補

初期のモデル別skill候補。

- GPT Image 2
- Nano Banana Pro
- Kling 3.0
- Seedance 2.0
- Runway Aleph / Runway系

このリストは固定ではない。画像生成、動画生成、3D、音声、編集、合成など、新しいモデルやサービスが増えたらskillを追加する。

## skillが扱う内容

モデル別skillには、少なくとも以下を含める。

- 公式ガイドラインや公開仕様から得られる注意点
- 社内検証で改善されたプロンプト構造
- 得意な表現
- 苦手な指定
- 必要な入力情報
- 入力画像 / 入力動画の扱い方
- 参照素材の使い方
- 推奨される生成設定
- ネガティブ指定や避けるべき表現
- 成功パターン
- 失敗パターン
- レビュー観点
- 情報源
- バージョン
- 最終確認日

公式ガイドラインやモデル仕様は変わるため、skillには必ず出どころと確認日を残す。社内検証で上書きした判断も、なぜその改善を入れたのかが追えるようにする。

## 呼び出しの考え方

PromptStudio は、ユーザーの意図と選択モデルを見てskillを呼び出す。

例。

- 商品ビジュアルを作るために GPT Image 2 向けskillを呼ぶ
- 参照画像を使ったルック検討で Nano Banana Pro 向けskillを呼ぶ
- カメラワークを含む動画生成で Kling 3.0 向けskillを呼ぶ
- ショート動画の動き検証で Seedance 2.0 向けskillを呼ぶ
- 映像表現の試作で Runway系skillを呼ぶ

## 保存する実例

モデル別skillを改善するには、結果の蓄積が必要である。

1回の相談では、以下を保存する。

- 制作意図
- 選択モデル
- 呼び出したskill
- skillの情報源
- skillのバージョン
- skillの最終確認日
- 参照したknowledge
- 入力画像
- 入力動画
- 参照素材
- 生成プロンプト
- 生成設定
- 生成結果
- 成功 / 失敗
- 失敗理由
- 次回改善案
- 管理者レビュー

## PDCA

### Plan

公式ガイドライン、公開仕様、社内検証、過去案件からskillを作る。

### Do

制作スタッフがPromptStudioで相談し、意図とモデルに合うskillを使ってプロンプトを作る。

### Check

管理者が、プロンプト本文だけでなく、入力素材、モデル選択、skill呼び出し、生成結果、成功 / 失敗理由まで確認する。

### Act

レビュー結果をモデル別skillと共通skillへ戻す。

## 重要な原則

PromptStudio は、各モデルを直接置き換えるものではない。

各モデルを仕事で使いやすくするために、会社の前提、公式ガイドライン、社内蓄積、レビュー結果をつなぐ共通レイヤーである。
