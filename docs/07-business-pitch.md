# ビジネスピッチ

## 要約

PromptStudio は、クリエイティブ制作現場の生成AI活用を、個人のプロンプト技術から、会社全体で再現・改善できる制作プロセスへ変えるための運用基盤である。

対象は、広告、映像、デザイン、CG / VFX、ゲーム、アニメ、ブランド制作、SNSコンテンツなど、生成AIでビジュアル、映像、企画、表現案を作るすべてのチーム。

目的は、全員の表現を平均化することではない。各クリエイターの個性や意図を活かすために、AIに渡す前提条件を揃えることである。

## なぜ今か

生成AIサービスは増え続け、制作現場で使える表現手段も広がっている。

一方で、ツールが増えるほど、現場には「どのサービスを、どんな指示で、どの案件条件に合わせて使うか」という運用課題が残る。

PromptStudio は、個別の生成AIサービスを置き換えるのではなく、会社が複数の生成AIサービスを再現性高く使うための共通レイヤーになる。

## 解く課題

生成AIの導入は進んでいるが、成果品質は担当者の言語化力、プロンプト経験、モデル理解、参照している前提に依存している。

多くの現場では、すでに社員が各自でAIチャットや画像生成・動画生成モデルを使い、プロンプトを組んでいる。課題は「AIにプロンプトを書かせれば解決する」ことではなく、AIに渡す会社の文脈、案件ルール、skill、モデル別ノウハウが人によって違うことにある。

そのため、以下が起きる。

- 出力品質が人によってばらつく
- 良いプロンプトが個人メモやチャット履歴に閉じる
- AIにプロンプトを書かせても、入力する前提が人によって違う
- 案件ルールやブランド文脈が毎回抜け落ちる
- 管理者がレビューしても、改善に必要な文脈が残っていない
- 会社として生成AIノウハウを蓄積できない

## 解決策

PromptStudio は、制作スタッフがChrome上で対話しながら、会社共通のskillとknowledgeを使ってプロンプトを作れるようにする。

同時に、会話、意図、参照情報、想定モデル、生成設定、最終プロンプト、レビュー用メモを保存し、管理者がskill改善へ戻せるようにする。

仕事でクリエイティブ制作の生成AIプロンプトを考えるときは、まず PromptStudio を使う。そこから、GPT Image 2、Nano Banana Pro、Kling 3.0、Seedance 2.0、Runway系など、意図とモデルに応じたskillを呼び出す。

## 価値

- 制作の土台になる前提条件を揃えられる
- 個性を消さずに、表現の土台になる前提条件を揃えられる
- 生成AIに強い人だけに依存しない
- 各自が違うAIモデルを使っていても、会社の前提を共有できる
- 案件やブランドの文脈を反映しやすい
- 公式ガイドラインと社内蓄積で改善されたskillを使える
- 成功 / 失敗、入力画像 / 入力動画、参照素材を蓄積し、共通skillを改善できる
- 良い実例と失敗例を会社の資産にできる
- 管理者がレビューし、skill改善のPDCAを回せる

## 最初の実装

最初はChrome拡張として、ChatGPTに近い対話UXを作る。

裏側ではローカルブリッジを通じて、workspace、skill、knowledge、成果物保存先、Google Antigravity / Antigravity IDE を扱えるようにする。

Antigravity を重視する理由は、Google公式プロダクトとして導入説明がしやすく、Googleアカウントと個人向け無料プランで始めやすく、Gemini系の高性能マルチモーダル理解、Projects、Skills、Artifacts、agent実行が PromptStudio の設計と合うためである。
