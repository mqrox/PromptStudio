# Product Brief

## 背景

広告、映像、デザイン、CG / VFX、ゲーム、アニメ、ブランド制作、SNSコンテンツなど、クリエイティブ制作の現場では、画像生成、動画生成、参照画像活用、スタイル設計、カメラワーク、ライティング、商品ビジュアル、コンセプト開発など、生成AIの使い道が増えている。

一方で、実際のプロンプト品質は社員ごとの経験、言語力、モデル理解、参照している前提に依存しやすい。生成AIに慣れている人は意図を細かく分解できるが、慣れていない人は「作りたいもの」は分かっていても、モデルが再現しやすい形に変換できない。

また、現場では各自が ChatGPT のようなAIチャットや、画像生成・動画生成の各種モデルを使ってプロンプトを組んでいる。課題は「誰もプロンプトを書けない」ことではなく、社員ごとに使うコンテキスト、知識、モデル前提、skillが違うことにある。

その結果、以下の問題が起きる。

- 同じ案件でも人によって出力品質がばらつく
- 画像生成と動画生成で必要な指定の違いが共有されない
- モデルごとの癖や制約が個人メモに閉じる
- 良いプロンプトが再利用されず、毎回ゼロから考える
- AIにプロンプトを書かせても、渡す前提が違うため出力が揃わない
- 管理者が品質改善のためにレビューしようとしても、会話や意図が残っていない

## 本質

PromptStudio の本質は、プロンプトを書くツールではない。

「AIにプロンプトを書いてもらえば各自うまく書ける」という問題設定でもない。AIが参照する前提、会社のknowledge、モデル別skill、案件ルールが揃っていなければ、AIが作るプロンプトも担当者ごとにばらつく。

**クリエイティブ制作会社の生成AIノウハウを、社員が日々使える対話UXと、管理者が改善できる保存形式に変換し、プロンプト作成の前提を組織として揃えること** が本質である。

これは、プロンプトを固定化して全員の出力を平均化することではない。各クリエイターの意図、ディレクション、個性を活かすために、案件ルール、モデル理解、入力素材の扱い方、失敗しやすい条件など、表現の土台になる前提条件を揃える。

社員は、ChatGPT のように自然に相談する。PromptStudio は、会社共通の skill、knowledge、案件文脈、モデル別ルールを使って、目的に合うプロンプトを組み立てる。

その過程と結果を保存することで、管理者は「なぜそのプロンプトになったか」「どのskillが効いたか」「何が足りなかったか」をレビューできる。レビュー結果は skill / knowledge に戻され、次の社員の出力品質が上がる。

仕事でクリエイティブ制作に関わる生成AIプロンプトを考えるときは、まず PromptStudio を使う。PromptStudio は、公式ガイドライン、社内検証で改善されたskill、過去案件の成功/失敗、入力画像や入力動画の利用情報を踏まえて、意図とモデルに合うプロンプト作成を支援する。

初期の実行基盤としては、Google Antigravity / Antigravity IDE を重視する。Googleアカウントで始めやすく、個人向け無料プランから検証でき、Gemini系のマルチモーダル理解、Projects、Skills、Artifacts、agent実行の考え方が PromptStudio の設計と噛み合うためである。ただし、Antigravity専用に固定せず、skill / knowledge / artifact は独立した資産として扱う。

将来的には、Chrome拡張だけでなく、CLI、MCP server、Local API から PromptStudio Core を呼べるようにする。これにより、ChatGPT系環境、Codex、Claude Code、Gemini / Antigravity、社内ツールなど、入口が違っても同じskill、knowledge、artifact、レビュー改善サイクルを使えるようにする。

## 対象ユーザー

### 制作スタッフ

- 画像生成や動画生成を業務で使う
- プロンプト設計の専門家ではない
- 日本語で作りたいものを相談したい
- モデルごとの差分や英語表現を毎回考えたくない
- 生成結果を案件やレビューに残したい

### 管理者 / AI推進担当

- 社内の生成AI利用品質を揃えたい
- 良いプロンプトや失敗例を蓄積したい
- 案件ごとのルールや禁止事項を反映したい
- 社員のアウトプットから skill を改善したい
- 会社として再現可能な制作フローにしたい

### 対象領域

- 広告制作
- 映像制作
- デザイン制作
- CG / VFX
- ゲーム / アニメ制作
- ブランド / マーケティング
- SNS / ショート動画制作
- 商品撮影、EC、ファッションビジュアル

## 解決したいこと

- 個人技になっているプロンプト設計を、会社の共通skillにする
- 社員ごとに違う前提、コンテキスト、モデル知識を仕組みで揃える
- 出力を平均化するのではなく、表現の土台になる前提条件を揃える
- 制作意図を、画像生成 / 動画生成に適した構造へ変換する
- モデル、案件、用途ごとの文脈を毎回入力しなくてよくする
- GPT Image 2、Nano Banana Pro、Kling 3.0、Seedance 2.0、Runway系など、さまざまなモデル別skillを呼び出せるようにする
- 成功、失敗、入力素材、生成結果を蓄積し、共通skill改善に使う
- プロンプトと副産物をレビュー可能な単位で保存する
- 管理者レビューから skill 改善へつなげる

## 作らないもの

初期段階では、以下を主目的にしない。

- 単なるChatGPTクローン
- 汎用メモアプリ
- 生成AIサービスの完全代替
- 個性を消すためのプロンプト規格化ツール
- すべてのモデルを直接実行する統合UI
- 管理画面だけが立派で、現場の入力体験が弱いツール

PromptStudio は、現場の対話UXと、管理者の改善サイクルの両方がそろって初めて価値を持つ。
