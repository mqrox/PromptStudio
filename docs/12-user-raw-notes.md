# User Raw Notes

## この文書の目的

この文書は、PromptStudio についてユーザーがチャットに貼った原文メモを、意味を変えずに一次情報として残すためのものである。

ここでの `raw文` は、要約や解釈ではなく、ユーザーが送った文章そのものを指す。誤字、表記ゆれ、言い回しも、意図の温度感を残すため原則そのまま保持する。

## Raw 01: 最初に実装したいもの

```text
Antigravity2.0IDEをエンジンとした、chrome拡張を実装したい。
やりたいこと。
VFX,CG制作会社で生成AIサービスのプロンプトを各々が考えている。
社員による知識や言語力によりプロンプトの再現性がない。画像生成の場合や動画生成の場合の各々のスキルがないので、スキルを共通化したい。
その上で各々やりたいことに応じて生成プロンプトを適切に考えてくれる。

そのプロンプトと副産物を任意のフォルダーに保存することで、管理者がそれをレビューし、skillを改善していくことができる。

まず実装したいのはchrome拡張としてchrome上でcahtgptなみのUXで対話操作ができること。
スキル、副産物の保存先、ローカルワークスペースのパス、knowledgeなどの共有コンテキストの場所。
が実装できる。それをAntigravityIDEが使えると良い。各々Googleアカウントでログインしてもらえるとさらに管理が助かる。

まず理解してほしい
```

## Raw 03: READMEの深さと対象業界

```text
何が課題なのか、何が解決されるのかなど。READMEが浅くならないようにしてほしい。このREADM.mdを社長がみてOKが出るレベルまで分かりやすく、ビジネスピッチレベルで仕上げてほしい。誰が見ても理解できること。またCG,VFXに限定されずクリエイティブにかかわる広告や制作などもふくめて業界全体でつかえるようなツールにしたい。
```

## Raw 04: AIにプロンプトを書かせるだけではない

```text
基本的には各々chatgptとか各々違うモデルとかでも組んでいる人もいるし、プロンプトを組んでいるんだけど、そのモデルも社員によってコンテキストやノウハウ、skillががうので、それを仕組みで解決しようって感じ。
前提が違うのでプロンプトをaiに書いてもらえば各々うまく書けるくない？ってそーゆー問題でもない。それも踏まえてほしい
```

## Raw 05: 仕事ではまずこれを使う

```text
とにかく、仕事でクリエイティブで生成aiでプロンプトを考える時はこれを使ってくれって感じ。
skillには意図に合わせて公式ガイドラインや蓄積で改善されたskillが呼び出される感じ。
gpt-image2で生成するとき、nanobanana proで生成するとき、kling3.0で動画生成するとき、seedance2.0で動画生成するとき、RunwayAlpheでやつときなどなど、さまざまな意図やモデルに対応させていきます。
どのようなときに失敗して、成功して、入力画像や動画になにをつかったのか、など共通skillを改善するための蓄積ができる。
```

## Raw 06: 平均化ではなく前提条件を揃える

```text
あくまでプロンプトをガチガチにしてみんあがみんな個性のない平均化された出力にすることが目的ではなく、あくまで底の前提条件をそろえることが本質。
```

## Raw 07: なぜAntigravity IDEなのか

なぜAntigravityIDEなのかも入れたい。
