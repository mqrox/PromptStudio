# Decision Log

## 2026-07-08: PromptStudio の初期方針を定義

### 決定

PromptStudio を、VFX / CG 制作会社向けの生成AIプロンプト運用基盤として定義する。

最初のUXは Chrome拡張のチャットUIとし、ローカルworkspace、skill、knowledge、成果物保存はローカルブリッジを経由して扱う。

Antigravity 2.0 IDE は実行エンジン候補として扱うが、プロダクト全体をAntigravity専用に固定しない。

### 理由

社員にとって一番重要なのは、普段のChrome上で迷わず相談できること。

一方で、任意フォルダー保存、workspace操作、IDE連携はChrome拡張だけで完結させると権限と保守性の問題が大きい。

そのため、Chrome拡張はUX、ローカルブリッジはファイルと実行環境、Antigravity 2.0 IDE は実行エンジンという責務分離にする。

### 影響

- MVPではChrome拡張とローカルブリッジを分けて実装する
- 保存形式は将来の管理者レビューとskill改善を前提に設計する
- Googleログインや権限管理は重要だが、初期はユーザー識別を記録できる設計に留める

## 2026-07-08: 保存対象をプロンプト本文だけにしない

### 決定

1回の相談ごとに、プロンプト本文だけでなく、依頼内容、対話ログ、参照skill、参照knowledge、生成設定、レビュー用メモを保存する。

### 理由

プロンプトだけが残っても、なぜその表現になったかが分からない。管理者がskill改善を行うには、意図、参照情報、失敗や迷いも必要になる。

### 影響

- 成果物フォルダーの基本構造を設計する必要がある
- Chrome拡張の保存ボタンは、単なるテキスト保存ではなく相談単位の保存になる
- 管理者レビュー用の `review.md` を初期から用意する
