# PromptStudio

> 生成AIサービスが増えるほど、プロンプトとノウハウは個人に散らばる。PromptStudio は、それを会社で蓄積・改善できる制作力へ変える。

PromptStudio は、生成AIを使うクリエイティブチームのための **プロンプト運用基盤** です。

社員は PromptStudio に「作りたいもの」を相談します。PromptStudio は、会社の制作ルール、案件ナレッジ、モデル別ノウハウ、過去の成功・失敗例を参照し、画像生成・動画生成・広告制作などで使えるプロンプトへ整理します。

保存するのは、プロンプトだけではありません。依頼内容、参照情報、入力素材、生成設定、結果、レビューコメントまで残します。管理者はその記録を見て、共通スキルを改善できます。

**目的は、全員の表現を平均化することではありません。各クリエイターの発想を活かすために、表現以前でズレやすい前提条件を会社として揃えることです。**

---

## 1. ひとことで言うと

PromptStudio は、仕事で生成AIプロンプトを考えるときに、最初に使う共通ワークベンチです。

ユーザーは「こういう絵を作りたい」「この商品を動画にしたい」「この案件のトーンで広告案を出したい」と日本語で相談します。

PromptStudio は、目的、媒体、モデル、入力素材、案件ルール、社内ノウハウを整理し、実際に使えるプロンプトと生成設定へ変換します。

そして、その過程を保存し、レビューし、次の制作に活かします。

---

## 2. 何を解決するのか

制作現場では、すでに多くの人が ChatGPT、画像生成AI、動画生成AIを使い始めています。

しかし、成果の品質は「誰がプロンプトを書いたか」に大きく左右されています。

同じ案件でも、担当者によって次のような差が出ます。

- 何をAIに伝えるべきか
- どのモデルを使うべきか
- そのモデルが得意な表現、苦手な表現は何か
- 案件ルールやブランドトーンをどう反映するか
- 入力画像や入力動画をどう使うか
- 失敗した条件を次にどう活かすか

この差は、単に「AIにプロンプトを書かせればよい」という話ではありません。

AIに渡す前提が人によって違えば、AIが作るプロンプトもバラつきます。会社のルール、案件文脈、過去の成功例、モデル別の注意点を知らないままでは、再現性のある成果にはなりません。

PromptStudio は、この問題を個人の努力ではなく、会社の仕組みとして解決します。

---

## 3. なぜ必要か

生成AIサービスは、今後さらに増えます。

画像、動画、3D、音声、編集、合成など、制作工程の多くにAIが入り込んでいきます。ツールが増えるほど、現場では「どれをどう使うか」「どう指示すれば狙った表現になるか」「案件ルールをどう守るか」が難しくなります。

このままでは、生成AIに強い一部の人だけが速くなり、会社全体の制作力にはなりません。

PromptStudio は、生成AI活用を個人の時短で終わらせず、会社の制作プロセスとナレッジ資産に変えます。

---

## 4. PromptStudio の仕組み

### 1. 案件のワークスペースを指定する

案件ごとに、制作フォルダ、前提条件となる資料ディレクトリ、入力素材の場所、成果物や副産物の保存先を指定します。

PromptStudio は「どの案件の文脈を参照するか」と「どこに記録を残すか」を明確にしたうえで、プロンプト作成を始めます。

### 2. 意図を整理する

専門的なプロンプト構文を知らなくても、日本語で相談できます。PromptStudio は、目的、構図、トーン、演出、制約条件を整理します。

### 3. 共通スキルと案件ナレッジを合わせる

スキルとは、単なるテンプレートではありません。

画像生成の構図、動画のカメラワーク、広告表現の注意点、ブランドトーン、モデル別の得意不得意、ネガティブプロンプト、レビュー観点などを含む、会社共通の制作ノウハウです。

そこに、案件ごとのブランドガイドライン、商品情報、世界観、禁止事項、キャラクター設定、過去の参考資料などを組み合わせます。

これにより、毎回ゼロから前提を説明しなくても、会社の共通スキルと案件固有の文脈を反映したプロンプトを作れます。

### 4. モデル別に最適化する

GPT Image 2、Nano Banana Pro、Kling 3.0、Seedance 2.0、Runway Aleph / Runway系など、用途やモデルに応じたプロンプトの作り方をスキルとして持ちます。

特定モデルに固定するのではなく、どのモデルを使う場合でも会社の前提条件を揃えて渡せるようにします。

### 5. プロンプトと副産物を保存する

最終プロンプトだけでなく、依頼、対話、参照したスキル、参照したナレッジ、入力素材、生成設定、結果、レビューコメントまで保存します。

保存先は案件やチームの運用に合わせて指定できるため、個人のチャット履歴ではなく、会社で見返せる共有資産として残せます。

これにより、「なぜそのプロンプトになったのか」「何を入力素材に使ったのか」「どの条件で成功・失敗したのか」を後から確認できます。

### 6. レビューして改善する

管理者は、保存された成果物を見て、何が良かったか、何が足りなかったか、どのスキルを直すべきかを判断できます。

改善されたスキルは次の社員の出力に反映されます。つまり、1回ごとのプロンプト作成が、次の制作を良くするための蓄積になります。

---

## 5. 導入すると何が変わるか

PromptStudio は、単にプロンプトを書く作業を助けるだけのツールではありません。

会社の生成AI活用を、教育可能で、改善可能で、再利用可能な制作プロセスに変えていきます。

| Before | After |
| --- | --- |
| プロンプト品質が担当者に依存する | 会社の前提条件を使い、品質のブレを減らせる |
| 良いノウハウが個人のチャット履歴に埋もれる | 成果物として保存され、再利用できる |
| 案件ルールを毎回説明する | 案件ナレッジとして参照できる |
| モデルごとの使い方が個人任せ | モデル別スキルとして共有できる |
| 新人や非専門者が使いこなしにくい | 会社のノウハウを使いながら相談できる |
| 管理者が改善点を追えない | 依頼、判断、結果、レビューを見て改善できる |
| 生成AI活用が個人の時短で終わる | 組織の制作プロセスとナレッジ資産になる |

---

## 6. 誰のためのツールか

### 制作スタッフ

作りたいイメージはあるが、プロンプト設計やモデルごとの書き方に自信がない人。日本語で相談しながら、案件ルールを外さず、使えるプロンプトへ整理できます。

### アートディレクター / クリエイティブディレクター

チーム全体の表現品質を底上げしたい人。良いプロンプトや失敗例を、個人のノウハウではなくチームの資産にできます。

### 管理者 / AI推進担当

生成AI活用を、個人の努力ではなく会社の運用にしたい人。利用データをもとに、教育、レビュー、ルール整備、スキル改善を回せます。

## 7. 対象領域

PromptStudio は CG / VFX に限定しません。

生成AIを使ってビジュアル、映像、企画、表現案を作るすべての制作チームを対象にします。

- 広告代理店
- 映像制作会社 / ポストプロダクション
- CG / VFX スタジオ
- デザイン会社
- ブランド / マーケティング部門
- ゲーム / アニメ制作
- SNS / ショート動画制作チーム
- 商品撮影、ファッション、ECビジュアル制作
- イベント、展示、空間演出の企画チーム

## 8. MVPで作るもの

最初は、Chrome拡張として動く対話UIを作ります。

ChatGPTに近い操作感で相談できますが、目的は汎用チャットではありません。会社のスキル、ナレッジ、制作フォルダ、保存先とつながり、制作現場で再利用・レビューできる形にします。

MVPでは次を実装します。

1. Chrome拡張のサイドパネルで対話できる
2. 共通スキル、案件ナレッジ、制作フォルダ、保存先を設定できる
3. 画像生成・動画生成・広告制作など、目的別にプロンプトを組み立てられる
4. モデル別スキルを呼び出せる
5. 会話、参照情報、プロンプト、設定、入力素材、結果を保存できる
6. 管理者がレビューし、スキル改善に使える

Googleアカウントログイン、チーム権限、管理ダッシュボードは重要ですが、MVPでは「誰が作った成果物かを記録できる設計」に留め、後から拡張します。

## 9. 保存される成果物

1回の相談を、1つのレビュー可能な成果物として保存します。

```text
artifacts/
  2026-07-08_153000_campaign-visual-idea/
    request.md
    conversation.md
    prompt.md
    prompt.json
    inputs.md
    result.md
    references.md
    metadata.json
    review.md
```

成功例だけでなく、失敗例、入力素材、生成設定、レビューコメントも残すことで、次の改善に使えます。

## 10. 初期の実行基盤について

PromptStudio は、画面上でプロンプトを作るだけのツールではありません。会社の制作フォルダ、スキル、ナレッジ、成果物、レビューをつなぎ、必要に応じてエージェントが制作支援を行える状態を目指します。

そのため、初期のローカル実行基盤として Google Antigravity / Antigravity IDE を重視します。

Google / Gemini 系の基盤であることは、制作会社や広告・映像・ブランド部門に導入するうえでも説明しやすいポイントです。Googleアカウントで始められ、画像や動画理解に強いGemini系のマルチモーダル性能とも近い位置にあります。

Google公式資料では、Antigravity 2.0 はエージェントを起動・監視・管理するための基盤として説明され、Antigravity、Antigravity IDE、CLI、SDK という構成が示されています。また、macOS、Linux、Windowsで利用でき、Googleアカウントでログインする流れも確認できます。

PromptStudio にとって重要なのは、次の点です。

- ローカル環境の制作フォルダを扱える
- エージェントが作業し、成果物を残せる
- スキルやナレッジという考え方と相性が良い
- MCPにより、外部エージェントや社内ツールと接続しやすい
- Googleのプロダクトとして、社内説明がしやすい

ただし、PromptStudio は Antigravity 専用にはしません。スキル、ナレッジ、成果物、レビューの仕組みは、将来的にほかの実行基盤にもつなげられるように設計します。

---

## 11. 将来像

PromptStudio の最初の入口は Chrome拡張ですが、本体は Chrome拡張だけではありません。

将来的には、Codex、Claude Code、Gemini、Antigravity、ChatGPT系環境など、どのエージェントからでも呼べる **MCPサーバー / 共通ハブ** にします。

どのAIツールを使う場合でも、PromptStudio が会社のスキル、ナレッジ、保存、レビューを提供し、プロンプト作成の前提条件を揃えます。

管理者向けには、蓄積された成果物を分析し、共通スキルの改善案を出す機能も用意します。ただし、共通スキルは自動で書き換えず、提案、レビュー、承認、反映を分けて運用します。

---

## 12. 目指す状態

PromptStudio が目指すのは、生成AIに強い一部の人だけが成果を出す状態ではありません。

クリエイティブチーム全体が、会社のノウハウを使って前提条件を揃えながら、それぞれの意図や表現の違いを活かしたプロンプトを作れる状態です。

そして、その結果を保存し、レビューし、次の制作に活かせる状態です。

生成AIを「便利な外部ツール」から、**会社の制作力を拡張する運用基盤** へ変えること。

それが PromptStudio の目的です。
